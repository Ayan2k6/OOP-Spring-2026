1. В чем основная разница между классом и интерфейсом? 
Основное отличие заключается в том, что класс определяет состояние (поля) и поведение (реализацию методов) объекта, являясь его полноценным чертежом. Интерфейс же задает строгий "контракт" — набор методов, которые класс обязан реализовать. Интерфейс описывает, что именно должен уметь делать класс, но не описывает, как он будет это делать (не считая default методов в современных версиях Java). Кроме того, на основе класса можно создать объект (используя new), а на основе интерфейса — нет.

2. Могут ли интерфейсы иметь поля? Если да, то каковы их неявные модификаторы? 
Да, интерфейсы могут иметь поля. В Java все переменные, объявленные внутри интерфейса, неявно получают модификаторы public, static и final. Это означает, что они автоматически становятся открытыми константами, которые принадлежат самому интерфейсу, и их значение нельзя изменить после инициализации.

3. Может ли класс реализовывать несколько интерфейсов? Объясните, почему это полезно. 
Да, класс в Java может реализовывать любое количество интерфейсов. Это крайне полезно, поскольку Java не поддерживает множественное наследование классов (один класс может расширять только один другой класс). Реализация нескольких интерфейсов позволяет разработчикам наделять объекты различными независимыми ролями и способностями. Например, как требуется в следующем задании, класс Student может одновременно быть и CanHavePizza, и Movable. Это делает архитектуру приложения гораздо более гибкой.
